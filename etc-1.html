[jekyll/jasper] Git 블로그 만들기

marline lee 10 Oct 2023 on Etc
해당 게시글이 알려줄 블로그 테마는 ✨jekyllt/jasper 입니다.

🛠순서

  1. Ruby를 다운로드 합니다.
  2. 테마 파일을 다운로드 받아 기본 구조를 만듭니다.
  3. Git Hub와 연동합니다.
    1. Branch 1과 연동합니다.
    2. Branch 2와 연동합니다.
  4. 기본 설정을 합니다.

🚗

1. Ruby를 다운로드 합니다.
rubyinstaller.org/downloads/


jekyll는 Ruby언어를 통해 개발한 프레임 워크입니다.
글쓴이는 "Ruby+Devkit 3.1.4-1"를 다운로드 했습니다. 너무 최신 버전을 다운로드하면 정상적인 동작이 힘들 수 있다고 하니 참고해 주세요 !

다운로드 후에는 Ruby cmd 창이 자동으로 열리는데, 이어서 다운로드를 받아줍니다.(이 과정을 무시하고 닫으면 설치가 되지 않은 채로 남습니다!)

그 후에 정상적으로 다운로드가 완료되었는지 터미널에서 확인합니다. 

  ruby -v

🚗 🚓

2. 테마 파일을 다운로드 받아 기본 구조를 만듭니다.

테마가 등록된 Git Hub[jekyllt/jasper]를 fork해도 되고, ZIP파일을 다운로드 받아 사용해도 됩니다. 이 글의 2~3번은 ZIP파일을 다운로드 받았을 경우의 순서입니다.

원하는 위치에 해당 압축 파일을 푼 후, cmd를 실행하여 번들러를 install 합니다. 

  cd [압축을 푼 폴더]
  gem install bundler #테마의 gem 파일 설치
  bundler install #관련 번들 설치
  bundle add webrick #webrick 설치
Ruby v3.0부터 gem에 webrick이 포함되지 않아 `require': cannot load such file -- webrick (LoadError) 라는 오류가 서버 기동 시 뜰 수 있습니다. 
  bundle exec jekyll s
실행하면 http://127.0.0.1:4000/jasper/에서 정상적으로 돌아가는 것을 확인할 수 있습니다.

🚗 🚓 🚕

3. Git Hub와 연동합니다.

🥃Branch 1과 연동합니다.

Git Hub에 저장소를 하나 추가합니다. 그리고 jasper-master과 연동 작업을 합니다. 

    git init
    git remote add origin [저장소 URL]
    git add .
    git commit -m 'init'
    git push origin [Branch 1]


🍺Branch 2와 연동합니다.

Git Hub에 저장소에 새로운 Branch를 추가합니다. 이름은 무난하게 'gh-pages' 로 짓습니다. (수정 시, 별도 작업 필요)
생성된 gh-pages는 [Branch 1]와 동일한 형태를 띄웁니다. 안을 비우고 서버 기동 시 생성된 jasper-pages를 넣어야 합니다. 

    cd ../jasper-pages/
    git init
    git remote add origin [저장소 URL]
    git remote update
그리고 jasper-pages 속 파일들을 전부 지우고 이어서 실행합니다. 
    git checkout gh-pages #원격 저장소의 파일들을 받았습니다.
한 번 더 jasper-pages 속 파일들을 전부 지우고 이어서 실행합니다. 
    git add .
    git commit -m 'init'
    git push origin gh-pages
이제 원격 저장소의 gh-pages는 깨끗합니다.
jasper-pages는 서버를 가동하면 생기는 폴더입니다. jasper-master가 아닌, jasper-pages를 읽어 화면에 뿌립니다.
그러므로 두 폴더는 한 쌍입니다. 단 한 폴더만 커밋하면 서비스 서버에서 정상적으로 돌아가지 않습니다.

🚗 🚓 🚕 🛺

4. 기본 설정을 합니다.

이제 github.io에 블로그를 보여주려면 작업해야 할 것이 있습니다.
jasper-master 폴더의 _config.yml과 _data안에 있는 authors.yml을 수정해야 합니다.
- _config.yml
Website의 정보를 원하는 방향대로 채워넣습니다.

name 블로그 이름
description 블로그 설명
short_url 서비스될 url
baseurl '/' 또는 '/[원하는 url]/'

username Git Hub 이름
repo 해당 원격 저장소 이름
branch [Branch 1] 명
relative_source ../[원격 저장소 명](추측)
destination ../jasper-pages
production_url 서비스가 될 Full url
source_url Git Hub 저장소 Full url
- authors.yml
작성자 정보를 기입합니다.
저장 후, bundle exec jekyll s 를 해주고 두 개의 폴더를 Git에 commit 합니다.
Github.io url에 들어가서 확인해요 !

후기💦

jasper은 현재 기준 v2가 최신이고, 이 글이 다루는 jasper은 2021년을 마지막으로 업데이트가 끝난 v1입니다. jekyll 테마가 여러 개 있는데, 제 눈에 띈 테마가 이거였네요..

마음에 드는 다른 테마가 있다면 시도해 보세요 ! jekyll이라면 비슷할 겁니다.
